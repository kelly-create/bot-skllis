# 🛡️ X账号操作安全规范

> ⚠️ 重要：遵循这些规则避免账号被限制或封禁

## 🚨 教训记录

### 2026-02-09 临时限制事件
- **原因**：短时间内操作过多（发帖+关注多个账号）
- **结果**：账号被临时限制
- **解决**：用户手动解除

## ⏰ 操作频率限制

### 发帖
| 类型 | 限制 | 建议 |
|------|------|------|
| 单次发帖 | 间隔 ≥ 5分钟 | 10分钟以上更安全 |
| 每日发帖 | ≤ 10条 | 3-5条为佳 |
| 带图发帖 | 间隔 ≥ 10分钟 | 避免连续发图 |

### 关注
| 类型 | 限制 | 建议 |
|------|------|------|
| 单次关注 | 1个/次 | 每次只关注1人 |
| 关注间隔 | ≥ 30秒 | 1-2分钟更安全 |
| 每日关注 | ≤ 20个 | 5-10个为佳 |
| 批量关注 | ❌ 禁止 | 分散到多天 |

### 点赞/转发
| 类型 | 限制 | 建议 |
|------|------|------|
| 点赞间隔 | ≥ 10秒 | 30秒更安全 |
| 每日点赞 | ≤ 50个 | 20-30个为佳 |
| 转发间隔 | ≥ 1分钟 | 5分钟更安全 |

## 🤖 反检测措施

### 必须遵守
1. **随机延迟**：每次操作前添加随机等待时间（3-10秒）
2. **人类行为模拟**：鼠标移动、滚动页面
3. **避免模式化**：不要固定时间操作
4. **分散操作**：不同类型操作交替进行

### 代码示例
```python
import random
import asyncio

# 随机延迟
async def human_delay():
    delay = random.uniform(3, 10)  # 3-10秒随机
    await asyncio.sleep(delay)

# 操作前调用
await human_delay()
await do_action()
```

## ⚠️ 禁止行为

1. ❌ 短时间内大量操作
2. ❌ 固定间隔的自动化操作
3. ❌ 批量关注/取关
4. ❌ 重复发布相同内容
5. ❌ 使用明显的自动化脚本
6. ❌ 频繁登录/登出

## ✅ 推荐行为

1. ✅ 模拟真人浏览（滚动、停留）
2. ✅ 随机化所有操作间隔
3. ✅ 每天操作时间不固定
4. ✅ 偶尔手动登录使用
5. ✅ 保持账号活跃但不过度
6. ✅ 关注真实感兴趣的内容

## 🔧 操作脚本规范

所有X自动化脚本必须：

```python
# 1. 导入随机模块
import random

# 2. 定义安全延迟函数
async def safe_delay(min_sec=3, max_sec=10):
    await asyncio.sleep(random.uniform(min_sec, max_sec))

# 3. 每次操作前调用
await safe_delay()

# 4. 限制单次操作数量
MAX_ACTIONS_PER_RUN = 3

# 5. 操作后等待
await safe_delay(10, 30)  # 操作后等待更长
```

## 📊 账号健康检查

定期检查：
1. 账号是否被限制
2. 帖子是否可见
3. 关注/粉丝变化是否正常

## 🆘 被限制后处理

1. **立即停止**所有自动化操作
2. **不要**尝试继续操作
3. **通知主人**手动解除
4. **等待**24-48小时后再尝试
5. **减少**后续操作频率
