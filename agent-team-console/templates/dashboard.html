新建任务
任务类型用于分类统计与筛选；执行角色用于标记责任归属（便于后续按角色追踪进度）。
先选模板，再填项目路径，点“一键填充”。不懂命令时优先用这个方式。
示例：cd /root/your-project && pytest -q ｜ cd /root/your-project && npm run build。 若你是新手，建议先使用上面的“业务模板”自动生成命令。
操作提示（中文）
  • 启动：把任务放入执行队列；并发满时会自动排队。
  • 停止：中止当前任务并标记为 failed（返回码 143/137）。
  • 重置：把 done/failed 任务恢复成 pending，便于二次执行。
  • 任务详情页：可看完整日志，页面每 4 秒自动刷新。
  • 状态说明：pending=待执行，running=执行中，done=成功，failed=失败。
